# complexity 评分 技术方案

## 1. 背景

PRD 要求基于文件大小和嵌套层级计算 0-100 的 complexity 分数。数据来自 files 表。

## 2. 方案

扣分制，100 分起。从 files 表聚合大文件占比、平均文件大小、最大文件大小，超过阈值扣分。

### 扣分规则

| 指标 | 阈值 | 扣分 |
|------|------|------|
| 大文件（>15KB）占比 | >20% | -25 |
| 大文件占比 | >40% | -45（替代） |
| 平均文件大小 | >10KB | -15 |
| 最大文件 | >50KB | -10 |

### API

`score::complexity(conn, snapshot_id) -> Result<i32>`

## 3. 关键决策

| 决策 | 结论 | 理由 |
|------|------|------|
| 大文件定义 | >15KB | 约等于 500 行代码 |
| 不用 AST | 启发式 | PRD 排除语言级分析 |
