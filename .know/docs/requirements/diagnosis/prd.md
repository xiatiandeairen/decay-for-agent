# 问题诊断

## 1. 问题

M3 给出了三维度健康分数，但分数只告诉用户"整体状态如何"，不告诉"具体哪里有问题"。用户看到 structural: 65 后还需要自己去分析哪些目录过深、哪些文件过大。需要自动从采集数据中识别具体问题并分级。

## 2. 目标用户

使用 decay 的 AI agent。agent 运行 `decay` 后获得具体问题列表，用于决定优先修什么。

## 3. 核心假设

**自动识别具体问题并分级 → agent 能直接知道"哪里有问题、多严重"，不需要自己分析原始数据。**

验证方式：运行 `decay`，输出包含分级问题列表（critical/warning/info）和具体文件/目录。

## 4. 方案

- **Before**: 只有分数，不知道具体问题在哪 → **After**: 输出具体问题列表，如"src/git.rs 是变更热点（1145 行 churn）"

## 5. 验收标准

- 运行 `decay` → 输出包含分级问题列表
- 每个问题包含：级别（critical/warning/info）、类别（structural/complexity/fragility）、具体文件或目录、量化数据
- 无问题时 → 输出"No issues found"
- 问题按级别排序（critical 先于 warning 先于 info）

## 6. 排除项

- 不包含重构建议（→ 重构处方 PRD）
- 不包含问题趋势对比（→ M5）
- 不包含自定义诊断规则
