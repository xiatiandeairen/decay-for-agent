# 文件结构扫描

## 1. 问题

项目结构性债务的核心信号之一是文件膨胀和目录层级失控——大文件、深嵌套、模块过度膨胀。当前没有自动化手段采集这些结构指标，无法量化"项目结构是否在恶化"。

## 2. 目标用户

使用 decay 的 AI agent。agent 运行 `decay` 后自动获得项目文件结构的量化数据，无需手动遍历。

## 3. 核心假设

**自动扫描文件树并采集结构指标 → 为 M3 的 structural 维度评分提供数据基础。**

验证方式：对真实项目运行 `decay`，SQLite 快照中有文件结构数据（文件数、目录深度、文件大小分布）。

## 4. 方案

- **Before**: 不知道项目有多少文件、目录多深、哪些文件过大 → **After**: 每次运行自动扫描文件树，结构指标写入快照

## 5. 验收标准

- 运行 `decay` → 扫描项目文件树（排除 .git、target 等）
- SQLite 快照包含：文件总数、目录总数、最大目录深度、各文件的路径和大小
- 对 1000+ 文件的项目运行 → 扫描完成，耗时 <5 秒

## 6. 排除项

- 不包含语言级解析（AST 分析、函数计数等 → M3 语言增强）
- 不包含 .gitignore 之外的自定义排除规则
- 不包含评分逻辑（→ M3）
