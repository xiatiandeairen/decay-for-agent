# 性能维度

## 1. 问题
嵌套循环、不必要的拷贝、同步阻塞调用等模式导致性能问题。现有维度不检测这些信号。

## 2. 核心假设
基于文件内容模式匹配检测常见性能反模式 → 用户能量化"代码有多少性能隐患"。

## 3. 指标
| 指标 | 检测方式 |
|------|---------|
| 嵌套循环密度 | 检测 for/while 嵌套 ≥3 层 |
| clone/copy 密度 | grep .clone()/.copy()/.deepcopy() |
| 同步阻塞调用 | grep sleep/time.sleep/thread::sleep/sync HTTP calls |
| N+1 查询模式 | 循环内 DB/API 调用模式 |

## 4. 验收标准
- performance 维度输出 0-100 评分 + 诊断 + 处方
- 大量嵌套循环 → 分数低 + 诊断指出位置
- 高 clone 密度 → 建议使用引用

## 5. 排除项
- 运行时性能分析（需要 profiler）
- 算法复杂度分析（需要 AST）
