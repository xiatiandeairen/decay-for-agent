# 重构处方 技术方案

## 1. 背景

PRD 要求为每个 critical/warning 问题生成可执行的重构建议。处方与诊断在同一模块中实现。

## 2. 方案

处方作为 Issue 的 prescription 字段，在诊断规则匹配时同步生成。Critical 和 Warning 级别必有处方，Info 级别无处方。

### 处方模板

| 问题类型 | 处方 |
|---------|------|
| 文件过多 | 按职责拆分为子模块 |
| 目录过深 | 扁平化嵌套目录 |
| 大文件 | 提取独立逻辑到新文件（附具体文件名） |
| churn 热点 | 拆分高频变更文件，隔离不稳定逻辑 |
| churn 集中 | 分散变更热点 |

## 3. 关键决策

| 决策 | 结论 | 理由 |
|------|------|------|
| 生成方式 | 规则驱动模板 | PRD 排除 LLM 生成 |
| Info 无处方 | 仅 critical/warning | 信息量太低不值得建议 |
