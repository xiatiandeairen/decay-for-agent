# 回归检测 — 自动标记显著分数下降

## 1. 问题

M2 提供了 velocity（长期趋势），但无法捕捉突发的分数下降。一次大规模重构或引入大量代码可能导致某个维度分数骤降，用户和 agent 需要即时发现这种回归。

## 2. 目标用户

- AI agent：回归标记触发优先级提升，自动关联相关 action
- 开发者：terminal/markdown 输出中醒目的回归警告
- v5 后续里程碑（M4-M6）：回归事件作为轨迹报告的组成部分

## 3. 核心假设

**相邻快照差值 > k × 历史标准差 → 准确标记回归，假阳性率 < 10%。**

验证条件：k=2 时，基于历史快照序列回测假阳性率。

## 4. 方案

- 复用 M1 的 `dimension_series()` 提取历史分数
- 计算相邻快照间差值的标准差，作为"正常波动"基线
- 最新分数下降幅度超过 k × σ 时标记为回归
- 区分 Moderate（> kσ）和 Severe（> 2kσ）两级严重度
- Report JSON 新增 `regressions` 字段
- terminal/markdown 渲染回归警告

## 5. 验收标准

- `std_dev()` 正确计算标准差，空序列返回 0.0
- `detect_regressions()` 对 ≥3 个快照的维度检测回归，<3 跳过
- k=2 默认值，Moderate 和 Severe 两级严重度
- 分数上升或持平不触发回归
- `--json` 输出包含 `regressions`（有回归时）
- terminal 输出回归警告行
- markdown 输出回归警告段落
- 新增测试：标准差计算、回归检测、无回归、不足快照、severe 级别
- `cargo test` 全部通过

## 6. 排除项

- 不做阈值预警（M4 负责）
- 不做跨维度回归关联（M5 相关性分析负责）
- 不做可配 k 值的 CLI 参数（内置 k=2，后续按需暴露）
- 不做回归自动修复建议（处方系统已覆盖）
