# structural 评分

## 1. 问题

M2 采集了项目文件结构数据（文件数、目录深度、文件大小），但原始数据无法直接回答"项目结构是否健康"。需要将结构指标量化为一个 0-100 的健康分，让用户和 agent 能直观判断结构层面是否在恶化。

## 2. 目标用户

使用 decay 的 AI agent。agent 运行 `decay` 后获得 structural 分数，用于判断是否需要结构层面的重构。

## 3. 核心假设

**基于文件结构指标计算 structural 分数 → 用户能量化"项目结构多健康"，识别目录层级失控和模块膨胀问题。**

验证方式：运行 `decay`，输出包含 structural: XX/100 分数。

## 4. 方案

- **Before**: 有文件结构数据但不知道好坏 → **After**: 输出 structural 0-100 分，分数越高越健康

## 5. 验收标准

- 运行 `decay` → 输出包含 structural 分数（0-100）
- 目录深度越深、文件数越多、顶层目录越臃肿 → 分数越低
- 对健康的小项目（<50 文件，深度 <3）→ 分数 >80

## 6. 排除项

- 不包含文件内容分析（函数数量、行数等 → complexity 维度）
- 不包含变更频率分析（→ fragility 维度）
- 不包含问题诊断和重构建议（→ M4）
