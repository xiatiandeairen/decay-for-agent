# structural 评分 技术方案

## 1. 背景

PRD 要求基于文件结构指标计算 0-100 的 structural 分数。数据来自 M2 的 files 表。

## 2. 方案

扣分制，100 分起。从 files 表聚合文件数、最大目录深度、顶层目录数，超过阈值扣分。

### 扣分规则

| 指标 | 阈值 | 扣分 |
|------|------|------|
| 文件数 | >500 | -20 |
| 文件数 | >1000 | -40（替代） |
| 目录深度 | >5 | -15 |
| 目录深度 | >8 | -30（替代） |
| 顶层目录数 | >15 | -15 |

### API

`score::structural(conn, snapshot_id) -> Result<i32>`

## 3. 关键决策

| 决策 | 结论 | 理由 |
|------|------|------|
| 算法 | 扣分制 | 简单直观，阈值可解释 |
| 阈值来源 | 经验值 | v1 先用固定阈值，后续可调 |
