# 测试加固

## 1. 问题

当前测试覆盖基本路径但缺少边界情况。`--debug` flag 不存在，调试时无法看到内部流程。

## 2. 目标用户

decay 的开发者和使用 `--debug` 排查问题的用户。

## 3. 核心假设

**补充测试 + --debug flag → 代码质量有保障，问题可快速定位。**

验证方式：`cargo test` 全通过，`decay --debug` 输出详细日志。

## 4. 方案

- **Before**: 测试不完整，无调试手段 → **After**: 边界测试覆盖，`--debug` 输出内部流程

### 任务

| 任务 | 文档 | 进度 |
|------|------|------|
| 测试加固 tech | [tech](tech.md) | 0/0 |

## 5. 验收标准

- `cargo test` → 全通过，覆盖采集层+分析层边界情况
- `decay --debug` → 输出详细日志（采集进度、评分计算、诊断过程）
- `decay`（无 --debug）→ 无额外日志输出

## 6. 排除项

- 不包含 CI 覆盖率报告
- 不包含集成测试（跨模块 E2E）
