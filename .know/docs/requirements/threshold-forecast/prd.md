# 阈值预警 — 基于趋势线性外推预测阈值突破

## 1. 问题

M2 提供了 velocity，M3 检测回归，但都是事后分析。用户和 agent 需要提前知道"按当前趋势，哪个维度会在什么时候跌破健康线"，以便提前干预。

## 2. 目标用户

- AI agent：基于预警制定长期重构计划，优先处理即将跌破阈值的维度
- 开发者：看到预警信息，了解健康趋势走向
- v5 M6 轨迹报告：预警作为轨迹报告的组成部分

## 3. 核心假设

**线性外推 + R² 置信度过滤 → 有意义的预警，避免噪声。**

验证条件：R² > 0.7 时才预警，≥5 个快照。

## 4. 方案

- 复用 M2 的线性回归斜率
- 新增 R² 计算，仅 R² > 0.7 且斜率为负时预警
- 线性外推：snapshots_until_breach = ceil((threshold - current) / slope)
- 健康阈值默认 60 分
- Report JSON 新增 `forecasts` 字段

## 5. 验收标准

- `r_squared()` 正确计算决定系数，< 2 个点返回 None
- `forecast_breaches()` 仅对 ≥5 个数据点、R² > 0.7、斜率 < 0 的维度预警
- 当前分数已低于阈值不预警（已经跌破了）
- `--json` 输出包含 `forecasts`（有预警时）
- terminal/markdown 展示预警信息
- 新增测试：R² 计算、正常预警、R² 过低不预警、斜率正不预警、已跌破不预警
- `cargo test` 全部通过

## 6. 排除项

- 不做非线性外推（线性足够，ML 留 v6+）
- 不做自定义阈值的 CLI 参数（内置 60，后续按需）
- 不做置信区间展示（R² 已足够过滤低置信预警）
