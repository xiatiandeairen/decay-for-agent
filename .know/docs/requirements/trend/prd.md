# 趋势追踪

## 1. 问题

单次运行的评分只反映当前状态，无法回答"比上次好了还是差了"。用户需要知道改动是否改善了项目健康。

## 2. 目标用户

使用 decay 的 AI agent。agent 运行 `decay` 后判断上次重构是否有效。

## 3. 核心假设

**自动对比上次快照 → 用户一眼看出变化方向，无需手动查历史。**

验证方式：第二次运行输出包含变化方向（↑↓→）。

## 4. 方案

- **Before**: 每次运行只有当前分数 → **After**: 有历史时自动附带变化方向

## 5. 验收标准

- 首次运行 → 无趋势输出
- 第二次运行 → 输出包含 ↑/↓/→ 和变化值
- 不同项目目录 → 各自独立对比

## 6. 排除项

- 不包含历史图表
- 不包含指定快照对比
