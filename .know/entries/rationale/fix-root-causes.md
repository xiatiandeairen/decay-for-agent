# 修根因不修表面

遇到多个相似问题时，先归类架构根因再设计通用方案，不逐个打补丁。

## 实例

v3 code review 发现 4 个 critical 问题，归类后是 3 个架构级根因：

| 表面问题 | 根因 | 通用修复 |
|---------|------|---------|
| 8 维度各自读文件（24000 次 I/O） | 无共享数据上下文 | DataStore 拉模型 |
| 重复日期逻辑 / 重复语言检测 | 无公共基础设施层 | util.rs 统一 |
| Windows 路径 / 初始 commit | 平台假设散落在业务中 | normalize_path 统一入口 |

## 判断标准

修 1 个 bug 还是修 1 类 bug？如果能归类 ≥2 个问题到同一根因，优先修根因。
