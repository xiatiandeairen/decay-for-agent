# 趋势分析算法选型：线性回归 + k×σ

## Why

trend/ 模块使用最小二乘线性回归 + k×σ 异常检测，而非更复杂的模型。

核心考量：
- **数据量小**：典型项目只有 5-20 个 snapshot，多项式/EWMA 容易过拟合
- **可解释性**：slope = "每 snapshot 变化 N 分" 直接可读，用户不需要理解模型
- **计算成本**：CLI 工具要求毫秒级响应，线性回归 O(n) 足够

## Rejected Alternatives

- **EWMA（指数加权移动平均）**：对趋势方向判断好，但不能给出 slope 数值，无法回答"变化多快"
- **多项式回归**：5-20 个点拟合 2 次以上几乎必然过拟合
- **时间序列分解（STL）**：需要季节性假设，健康分数无周期性

## Constraints

- velocity: slope > 1.0 = Improving, < -1.0 = Declining — 阈值 1.0 是"1 分/snapshot"，低于此视为噪声
- regression: k=2.0 意味着偏离均值 2 个标准差触发检测 — 统计学常规 2σ 原则
- forecast: R² > 0.7 才做预测 — 低于此线性关系不够强，预测无意义，0.7 是统计学中等相关门槛
- forecast 要求 ≥5 点而非 velocity 的 ≥3 点 — 预测需要更多数据点才有信度
