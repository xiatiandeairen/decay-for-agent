# 拉模型优于推模型

共享数据时用 DataStore + OnceCell（拉模型），不用预加载 Context（推模型）。

## 对比

| | 推模型（AnalysisContext） | 拉模型（DataStore） |
|---|---|---|
| 新增数据源 | 改 ctx 结构 + 构建处 | 加 OnceCell 字段 + getter |
| 内存 | 全量预加载 | 按需加载，谁用谁付费 |
| 未用数据 | 也会加载 | 不调用不加载 |
| 框架改动 | 每次加数据源改框架 | 只加 getter |

## 判断标准

好的架构 = 新增功能只加文件不改框架。如果新增一种数据需要修改 trait 签名或框架调度代码，说明抽象层级有问题。
