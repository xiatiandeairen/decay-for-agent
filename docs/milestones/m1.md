## 1. 目标

### Definition of Done

tree-sitter-rust 集成 + 4 个 metric 计算 + 函数指纹 + SQLite 快照存储 + diff 引擎 + CLI（`decay` / `decay diff`）端到端可运行; 在 decay 自身代码上完整跑通无 panic, 4 个 metric 在抽样函数上人工验证结果合理。

### 价值目标

作者在 decay 项目运行 `decay` 创建首次快照, 改完代码后运行 `decay diff` 立即看到哪些函数 metric 上升及上升幅度。

### 关键假设

- **假设**: tree-sitter-rust 在 decay 自身代码（含 macro / async / trait / impl 块）上能完整解析无 panic, 4 个 metric 算法在 Rust 特殊语法（match arm / `?` 操作符 / closure）下输出合理结果
- **类型**: technical feasibility
- **验证方式**: 在 decay 自身代码运行 `decay`, 100% .rs 文件解析成功（解析失败计为关键假设不成立）, 抽样 5-10 个函数人工核对 4 个 metric 输出与手算结果一致

## 2. 计划

### 计划交付物

- code: tree-sitter-rust 集成 + 函数提取（含 fn / impl method / trait method）
- code: 4 个 metric 计算器（嵌套深度 / 圈复杂度 / 认知复杂度 / 参数数）
- code: 函数指纹生成（基于 file_path + name + param_types）
- code: SQLite 快照存储（snapshots + functions 两张表）
- code: diff 引擎（按 signature_hash 对齐 + delta 计算 + 排序输出）
- code: CLI 入口（`decay` / `decay diff` / `--help` / `--version`）
- test: 4 个 metric 单元测试 + 函数指纹稳定性测试 + diff 引擎集成测试
- data: decay 自身项目的首次完整快照（基线数据）

### Go/No-Go 标准

| 维度 | 标准 |
|------|------|
| 功能完整性 | 4 个 metric 全部实现可用; CLI `decay` 和 `decay diff` 都可调用; diff 引擎按 signature_hash 正确对齐函数; 首次运行（无 baseline）有友好提示不报错 |
| 质量 | 4 个 metric 单测全部通过（基于业界经验已知样例, 含正常 / 边界 / 极值）; tree-sitter 在 decay 自身 ≥30 个 .rs 文件上 100% 解析成功（解析失败的文件 warning 跳过, 不中断整体） |
| 性能与稳定性 | 在 decay 自身（估算 ~30 文件 ~6000 行 Rust）扫描 + 计算 + diff 端到端 <5 秒（目标值, 待验证, 基于经验估算） |
| 安全 | 不适用（CLI 工具仅本地文件读取 + 本地 SQLite, 无外部输入 / 网络 / 凭据） |
| 可维护性 | 不适用（v0.1 早期阶段, 维护性指标随项目演进决定） |

### 预估工时

~8 天 单人全职（估算, 基于工作分解: 解析 2 + metric 2 + 存储 1 + diff 1 + 测试 2）

### 决策路径

- **Pass**: 进入 v0.1 dogfood 验证阶段（人工抽查 5-10 函数 metric 合理性 + 故意制造退化验证 diff 准确性 + 持续 dogfood 至少 1 次"工具发现退化的瞬间"）; 同时基于实际跑通经验决定 v0.2 多语言扩展节奏
- **Fail**: 排查具体阻塞点（tree-sitter 解析失败 / metric 算法错误 / 函数指纹不稳定 / diff 错误对齐）, 不进入 dogfood 阶段, 必要时回审 v0.1 范围

### 风险

Dependencies: tree-sitter-rust crate 在 decay 项目使用的 Rust 语法范围（含 macro / async / trait / impl）内完整可用; rusqlite crate 跨平台兼容（macOS / Linux）。 Risks: 认知复杂度公式在 Rust 特殊模式（match arm 多分支 / `?` 操作符链）上的阈值需校准, 可能在 dogfood 后调整; 函数指纹在函数重命名 / 跨文件移动场景必然误报"删除+新增", v0.1 接受此 trade-off。
