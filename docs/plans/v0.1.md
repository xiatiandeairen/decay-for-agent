## 2. 共享契约

### 2.1 模块结构

```
src/
  main.rs         // bin entry
  lib.rs          // re-exports public API
  types.rs        // 共享类型 (无依赖)
  error.rs        // DecayError + Result (无依赖)
  config.rs       // Thresholds + 常量 (无依赖)
  walk.rs         // 文件遍历
  parser.rs       // tree-sitter 集成 + 函数提取
  fingerprint.rs  // 函数指纹
  metric/
    mod.rs        // pub fn compute(...) -> Metrics
    nesting.rs
    cyclomatic.rs
    cognitive.rs
    params.rs
  pipeline.rs     // walk + parser + metric + fingerprint orchestrator
  store.rs        // SQLite 读写
  diff.rs         // 快照对比
  cli/
    mod.rs        // clap 定义 + run()
    scan.rs       // decay 命令
    diff_cmd.rs   // decay diff 命令
```

### 2.2 依赖方向（DAG）

```
types       :  (无)
error       :  (无)
config      :  (无)
walk        :  error
fingerprint :  types
parser      :  types, error  + tree_sitter
metric/     :  (无内部依赖)  + tree_sitter
pipeline    :  walk, parser, metric, fingerprint, types, error
store       :  types, error  + rusqlite
diff        :  types, config
cli/        :  pipeline, store, diff, types, error, config  + clap
main        :  cli, error
lib         :  re-export
```

任何反向依赖 = 契约违反, 退回 subagent 修复。

### 2.3 核心类型（types.rs）

```rust
use serde::{Deserialize, Serialize};

#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct Function {
    pub file: String,             // 项目根相对路径, 正斜杠
    pub name: String,             // fn 名, 不含 impl receiver
    pub start_line: u32,          // 1-indexed
    pub end_line: u32,            // 1-indexed
    pub param_types: Vec<String>, // 归一化后, 用于指纹
    pub signature_hash: u64,      // fingerprint::compute() 结果
    pub metrics: Metrics,
}

#[derive(Debug, Clone, Copy, PartialEq, Eq, Serialize, Deserialize)]
pub struct Metrics {
    pub nesting: u32,
    pub cyclomatic: u32,
    pub cognitive: u32,
    pub params: u32,
}

#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct Snapshot {
    pub id: i64,
    pub project_id: String,       // 项目根绝对路径
    pub created_at: i64,          // unix timestamp seconds
    pub functions: Vec<Function>,
}

#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct DiffEntry {
    pub function: Function,       // 当前快照里的函数
    pub previous: Option<Metrics>, // None = 新增
    pub kind: DiffKind,
}

#[derive(Debug, Clone, Copy, PartialEq, Eq, Serialize, Deserialize)]
pub enum DiffKind {
    Added,
    Worsened,
    CrossedThreshold,
}
```

### 2.4 SQLite Schema（store.rs）

```sql
CREATE TABLE IF NOT EXISTS snapshots (
    id           INTEGER PRIMARY KEY AUTOINCREMENT,
    project_id   TEXT NOT NULL,
    created_at   INTEGER NOT NULL
);
CREATE INDEX IF NOT EXISTS idx_snap_project ON snapshots(project_id, id DESC);

CREATE TABLE IF NOT EXISTS functions (
    snapshot_id      INTEGER NOT NULL REFERENCES snapshots(id) ON DELETE CASCADE,
    signature_hash   INTEGER NOT NULL,
    file             TEXT NOT NULL,
    name             TEXT NOT NULL,
    start_line       INTEGER NOT NULL,
    end_line         INTEGER NOT NULL,
    nesting          INTEGER NOT NULL,
    cyclomatic       INTEGER NOT NULL,
    cognitive        INTEGER NOT NULL,
    params           INTEGER NOT NULL,
    PRIMARY KEY (snapshot_id, signature_hash)
);
CREATE INDEX IF NOT EXISTS idx_func_snap ON functions(snapshot_id);
```

DB 路径: `dirs::data_dir().join("decay").join("snapshots.db")`。
测试覆盖路径: 环境变量 `DECAY_DB_PATH`（仅测试用, 文档不公开）。
`u64 → i64` bit-cast 存储, 读取时反向 bit-cast。

### 2.7 错误类型（error.rs）

```rust
use thiserror::Error;

#[derive(Error, Debug)]
pub enum DecayError {
    #[error("io error at {path}")]
    Io { path: String, #[source] source: std::io::Error },

    #[error("parse error in {path}: {message}")]
    Parse { path: String, message: String },

    #[error("database error: {message}")]
    Db { message: String, #[source] source: rusqlite::Error },

    #[error("no snapshots for this project — run `decay` first to create one")]
    NoSnapshots,

    #[error("invalid project: {0}")]
    InvalidProject(String),
}

pub type Result<T> = std::result::Result<T, DecayError>;
```

策略:
- 单文件解析失败 → pipeline 层 catch, log warn, 跳过继续
- DB 错误 → 传播到 cli, 退出码 1
- NoSnapshots → 转为友好提示, 退出码 0

### 2.8 CLI 行为 + 输出 schema

`decay`（默认）:
- 扫描 cwd → 创建快照 → 显示**当前**超阈值函数
- 不带 baseline 时给"first snapshot"提示
- 输出格式（terminal）:

```
decay v0.1
Scanned {N} files, {M} functions in {T}s

Snapshot #{id} saved [first snapshot — run `decay diff` after next change]

{K} functions exceed threshold:

  {file}:{line}  {name}
    {metric}: {value} ⚠ (>{threshold})
    ...
```

`decay diff`:
- 对比最近两个快照, **不创建新快照**
- 无 baseline → "Run `decay` to create a baseline"
- 输出格式:

```
decay v0.1
Diff: snapshot #{curr} vs #{prev} ({elapsed} ago)

{K} functions degraded:

  {file}:{line}  {name}  [new|worsened]?
    {metric}: {prev} → {curr}  (+{delta}) {⚠ crossed | ⚠ already over}?
    ...
```

无变化:

```
decay v0.1
Diff: snapshot #{curr} vs #{prev}

✓ No functions degraded since last snapshot.
```

退出码（v0.1 简化）:
- `0`: 任何正常运行（含友好提示）
- `1`: panic / 致命错误 / DB 错误

### 2.9 Diff 报告策略

进入报告的函数:
| Kind | 条件 | 标签 |
|------|------|------|
| `Added` | 新增函数 **且** 当前任一 metric ≥ 阈值 | `[new]` |
| `CrossedThreshold` | 旧值 < 阈值, 新值 ≥ 阈值 | `crossed` |
| `Worsened` | 旧值已 ≥ 阈值, 新值更高 | `[worsened]` |

不进入报告: 阈值以下变动 / 函数下降 / 函数被删除 / 未变化函数。

排序: `max(metric_value - threshold)` 降序。

### 2.10 函数指纹算法（fingerprint.rs）

⚠️ 必须用确定性 hasher。`std::DefaultHasher` 跨进程不稳定, **禁用**。

```rust
use xxhash_rust::xxh3::xxh3_64;

pub fn compute(file: &str, name: &str, param_types: &[String]) -> u64 {
    let mut bytes = Vec::with_capacity(128);
    bytes.extend_from_slice(file.as_bytes());
    bytes.push(0);
    bytes.extend_from_slice(name.as_bytes());
    bytes.push(0);
    for t in param_types {
        bytes.extend_from_slice(t.as_bytes());
        bytes.push(0);
    }
    xxh3_64(&bytes)
}
```

参数类型归一化（T3 parser 必须遵守）:
- 去除所有空白
- 去除 lifetime（`&'a str` → `&str`）
- 保留泛型 / 引用 / 可变性
- self / `&self` / `&mut self` 三种字面字符串

### 2.11 函数提取规则（parser.rs）

| tree-sitter 节点 | 提取 | 备注 |
|----------------|------|------|
| `function_item` | ✅ | 普通 fn, impl 方法, trait 默认实现 |
| `function_signature_item` | ❌ | 无函数体 |
| `closure_expression` | ❌ | 内部代码计入外层 fn |
| 宏内的 fn | ❌ | 不可靠 |
| 文档测试 fn | ❌ | 不在主源 |

同名同参不同 impl: 共享指纹（v0.1 接受偏差, 实际罕见）。

### 2.12 认知复杂度公式（metric/cognitive.rs）

参考: SonarSource 白皮书。形式化伪代码:

```
cognitive(node, nesting=0):
    score = 0
    for child in node.children:
        if child.kind in {if_expression, while_expression, for_expression, loop_expression, match_expression}:
            score += 1 + nesting
            score += cognitive(child.body, nesting + 1)
        elif child.kind in {else_clause}:
            score += 1
            score += cognitive(child, nesting + 1)
        elif child.kind == match_arm (除第一个 arm):
            score += 1
        elif child.kind in {logical_and, logical_or}:
            // 同链内只 +1, 不重复
            score += 1 if first_in_chain else 0
        elif child.kind == try_expression (即 ? 操作符):
            score += 1
        elif child.kind == closure_expression:
            score += 1 + nesting
            score += cognitive(child.body, nesting + 1)
        elif child.kind in {return, break, continue} and inside_loop_or_branch:
            score += 1
        else:
            score += cognitive(child, nesting)
    return score
```

简化: 不检测递归; `?` chain 不去重; 不处理宏内。

- 所有单测通过

### T4 — Fingerprint

**目标**: 按 §2.10 实现确定性 hasher。

**必读**: §2.1, §2.10

**输出**:
- `src/fingerprint.rs`: compute(file, name, param_types) -> u64
- `tests/fingerprint_test.rs`:
  - 同输入 → 同输出
  - 字段顺序敏感（file/name/params 不能交换）
  - 跨"模拟进程"稳定（用已知输入对照 xxh3_64 文档值, 或用预计算的 hex 串）
  - 参数顺序敏感（[A, B] != [B, A]）
  - 空 param_types

**验收**: 所有单测通过 + clippy 0 警告。

**不做**: 调用图 / 跨文件追踪。

### T6 — Metric: Cyclomatic

**目标**: 圈复杂度 = 分支数 + 1（if/match arm/while/for/loop/&&/||/?）。

**必读**: §2.1, §2.3, §2.5, §2.11

## 6. 合并协议

### 派发流程

parent agent 用 `Agent({isolation: "worktree", subagent_type: "general-purpose", prompt: <§4 模板填充>})`, branch 命名 `task/T{N}-{slug}`。

### 合并顺序

```
W1 (T1) → fast-forward → cargo build 验证
W2 (T2, T3, T4) 并行 → 顺序 rebase 合 → 每次 rebase 后 cargo build && cargo test
W3 (T5-T8) 并行 → 顺序 rebase 合 → 验证
W4 (T9, T10) 并行 → 顺序 rebase 合 → 验证
W5 (T11) → rebase 合 → cargo test 全通过
T12 由 parent agent 直接做
```

任一 wave 验证失败 → 该 task 退回 subagent 修复, 不强行进入下一 wave。

### Conflict 预案

- 同 wave 两 task 改同一文件 → 不应发生（owner 制）。如发生, 保留先合的, 后合的人工 rebase
- subagent 标 BLOCKED → parent agent 升到主对话, 用户决策, 更新 §2 后重派
- 测试冲突: tests/ 下每个 task 独立测试文件, 不共享

## 7. 质量门（三层）

### 任务级（subagent 自验, 不通过不交付）

- `cargo build` 通过
- `cargo clippy -- -D warnings` 0 警告
- `cargo test --test {本 task 测试}` 全通过
- 输出文件清单符合 §5 brief
- 公共 API 签名严格符合 §2.5

### Wave 级（parent agent 合并后验）

- `cargo build` 通过
- `cargo test` 全通过（含已合并的所有 task 测试）
- 模块依赖方向无反向（手工抽查 / 工具：cargo modules tree）

### v0.1 级（user 验收）

- T12 audit.md 通过 user review
- dogfood 中至少 1 次"工具发现退化的瞬间"被记录（**唯一 hard gate**）

## 8. 风险与回退

| 风险 | 触发信号 | 回退动作 |
|------|---------|---------|
| tree-sitter-rust 解析 decay 自身 panic | T3 测试或 T12 audit 阶段 | 标 BLOCKED, 升级 user 评估是否 fallback 到正则 |
| 认知复杂度公式跑出明显不合理结果 | T7 5 样例误差 >1 / T12 抽查不合理 | 砍 cognitive metric, v0.1 仅 3 metric, 文档记录 |
| 函数指纹方案不稳定 | T4 测试或 T11 集成测试 | 退化为 file:name 简化, 接受重命名误报 |
| dogfood 8 天没出现"工具捕捉退化的瞬间" | T12 后持续 dogfood | 暂停 v0.1, 回审定位是否成立 |
| Subagent 输出严重偏离契约 | 合并时验证失败 | 退回 subagent 修复, 不擅自补 |

## 9. 验收协议（user 视角）

按顺序验收:

1. **看 §2 兑现**（T12 audit.md 自动汇总）—— 任一块未兑现, v0.1 不验收
2. **跑端到端命令**（user 自己跑）:
   ```
   git checkout main && cargo build
   cd /path/to/some/rust/project
   decay
   decay diff
   ```
   行为符合 §2.8 schema
3. **dogfood 验证**（user 持续使用, 不少于 3 天）
4. **唯一 hard gate**: 至少 1 次"工具发现退化的瞬间"被作者记录, **作者本人判定**
